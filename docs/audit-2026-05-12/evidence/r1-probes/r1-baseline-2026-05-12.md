## Findings (raw observations to convert to F-#### later)

1. **Stellar-core still bound to 11726** despite CLAUDE.md "stellar-rpc is NOT in our production ingest path … Removed from r1 on 2026-04-23 alongside stellar-core". (→ EV-1204; see W21 / R12; needs F-#### finding.)
2. **Removed `/v1/coins/*` routes still serving 200s on R1.** Commit `28ac6ac9` removed them in rc.48 (just tagged), but the running binary predates the deploy. Verify when the new binary lands; if persistent, finding. Live latency for these calls is 1.6-1.7 s — slow. (→ EV-1205; see W11 / R12.)
3. **MinIO at 78% disk usage (4.9T/6.4T).** Verify alert exists for >85% threshold; verify retention policy. (→ EV-1206.)
4. **Memory at ~95% with swap fully consumed.** Investigate top consumers (likely MinIO buffer + stellar-core). Risk of OOM during traffic spike. (→ EV-1206.)
5. **`/opt/ratesengine/scripts/dev/r1-smoke.sh` does not exist** but `ratesengine-smoke.timer` is firing. Need to read the smoke service unit to find the actual invoked path. (→ EV-1207; see W14 / J22.)
6. **`/v1/price` 404s take 200-300 ms.** Sub-path slow lookup before producing the not-found envelope. Investigate cache miss + DB lookup vs early-out. (→ W11.)
7. **`/v1/assets/{id}` cache-miss path takes 400-700 ms** for several asset IDs. Investigate prewarmer coverage and SEP-1 fetch latency. (→ W11 / W12.)
